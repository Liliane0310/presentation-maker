# Style Tokens

`presentation-maker` 的默认视觉系统以 `Workflow入门.html` 为母版：纸感、暖调、单页卡片式演示。它服务于“讲给人听”的教学演示，不追求炫技，也不切换成强烈科技风。

## 设计目标

- 背景像温和的纸张，不像发光面板
- 每一页像一张被放在桌面上的纸卡，而不是整屏应用界面
- 组件有轻微卡片感，但不过度立体
- 强调色只服务于记忆点、引导和结构分层

## 核心 CSS Tokens

```css
:root{
  --paper: #fffdf8;
  --ink: #1f2937;
  --muted: #5b6470;
  --accent: #8f4d2e;
  --accent-soft: #f3e6da;
  --line: #d8d0c4;
  --shadow: 0 24px 60px rgba(88,59,34,.12);
}
```

## 字体策略

- 中文正文优先：
  - `"XiaoLai"`
  - `"Noto Sans SC"`
  - `"Microsoft YaHei"`
  - `sans-serif`
- 中文标题优先：
  - `"XiaoLai"`
  - `"Microsoft YaHei"`
  - `sans-serif`
- 等宽字体优先：
  - `ui-monospace`
  - `"SFMono-Regular"`
  - `"Consolas"`
  - `monospace`

如果本地没有 `XiaoLai`，允许自动回退到更稳妥的系统字体，不要求强绑外部字体文件。

## 类型层级

- `h1`
  - 2.8rem-3.4rem
  - 用于封面和核心结论页
- `h2`
  - 1.7rem-2rem
  - 用于常规章节页标题
- `big`
  - 1.25rem-1.35rem
  - 用于关键导语或长句提示
- 正文
  - 1rem-1.08rem
  - 行高 1.75-1.85
- 辅助信息
  - 0.85rem-0.95rem
  - 颜色使用 `--muted`

## 组件使用规则

### 背景

- 页面底色优先使用暖纸色和轻微径向或线性渐变
- 允许叠加轻微纸张纤维纹理
- 不使用高对比黑底荧光色作为默认风格

### 边框和阴影

- Slide 本体需要有明确纸张边界、圆角和柔和阴影
- 卡片允许有轻边框和柔和阴影
- 阴影要克制，不能出现厚重悬浮感
- 表格和信息块优先用细边框组织结构

### 强调色

- `--accent` 用于重点词、二级标题、流程节点、关键结论
- `--accent-soft` 用于浅底标签、代码底、说明底色
- `--muted` 用于页码、辅助说明、次级段落

不要把强调色铺满整页，也不要让整页看起来像广告图。

## 推荐组件气质

- `blockquote`
  - 像讲义里的摘录框
- `card`
  - 像便于扫读的小知识卡
- `flow`
  - 像手绘流程图
- `table`
  - 像整理过的课堂笔记
- `code` / `kbd`
  - 仅在确实需要展示命令、代码、快捷键时使用

## 禁止项

- 不切换成完全不同的视觉主题
- 不默认使用深色主题
- 不引入高饱和科技蓝、霓虹紫、赛博风
- 不使用大面积玻璃拟态或过度动画
- 不让页面像产品 landing page，而要更像“好讲、好读、好回看”的纸张式 deck
