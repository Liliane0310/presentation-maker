# Script Guide

这个文件只管 `.notes`，不管视觉布局。

目标不是写论文，而是写“讲起来顺”的演讲辅助稿。

## 三种 notes 模式

### 1. 完整逐字稿

默认模式。用于用户希望“几乎能直接照着讲”的场景。

规则：

- 每页约 150-300 字
- 语气口语化
- 至少有一个开头钩子
- 说明当前页的核心点
- 最后最好能过渡到下一页

适合：

- 教学分享
- 新手科普
- 演讲者需要完整安全感的场景

### 2. 关键提示词

轻量模式。用于用户熟悉内容，只需要提醒顺序。

规则：

- 每页 3-5 条
- 每条只说一个动作或一个重点
- 允许写成讲述顺序提示，而不是完整句子

建议格式：

```html
<div class="notes">
  <p>1. 先抛问题，让听众进入情境。</p>
  <p>2. 用一个生活类比解释核心概念。</p>
  <p>3. 强调这页只要记住一个判断。</p>
</div>
```

### 3. 不要 notes

适用于用户明确不需要演讲者稿的情况。

规则：

- 可以省略 `.notes`
- 或保留空的 `.notes`

## 完整逐字稿规范

### 开头钩子

每页尽量以一个能把人拉进来的句子开场：

- 问题开场
  - “你们有没有发现……”
  - “为什么这个东西听起来难，其实又离我们很近？”
- 故事开场
  - “我第一次碰到这个概念的时候……”
- 数据开场
  - “这里有一个很能说明问题的数字……”

避免：

- “接下来我们讲……”
- “这一页展示了……”
- “如图所示……”

### 口语化

推荐：

- “所以” 代替 “因此”
- “这个东西” 代替 “该对象”
- “说白了就是” 代替 “其本质在于”
- “你可以把它理解成” 代替 “其定义为”

不推荐：

- 书面论文腔
- 官样汇报腔
- 纯摘要罗列

### 结构建议

每页完整逐字稿可以按这个内在顺序组织：

1. 先用一句钩子把人拉进来
2. 再解释这页的核心概念
3. 如果有术语，立刻白话解释
4. 如果概念抽象，补一个生活类比
5. 最后给一句过渡或收束

## 提示词模式规范

提示词不是摘要，而是“讲述动作提示”。

推荐写法：

- 先指出听众常见误解
- 再给白话定义
- 再补一个生活类比
- 最后用一句总结收回来

不推荐写法：

- “定义、特点、意义、应用”
- “背景、问题、方案、结论”

后者太像写提纲，不像演讲提示。

## 解释优先规则

- 新术语首次出现必须加白话解释
- 抽象概念优先给生活类比
- 如果听众是新手，先讲“它像什么、它解决什么”，再讲“它正式叫什么”

## 不推荐表达

- “这一页展示了……”
- “接下来我们讲……”
- “综上所述……”
- “如前文所述……”
- “本文将介绍……”

这些说法太像报告，不像现场讲述。
