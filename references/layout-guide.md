# Layout Guide

这个文件负责回答一个问题：某类内容，应该放进什么页面结构里。

生成时，优先按 `Workflow入门.html` 这一类单页纸张母版来选布局，而不是先想花样。

## 场景到结构映射

### 1. 封面页

- 推荐结构：`section.slide`
- 适用内容：
  - 标题
  - 副标题或导语
  - 一句总纲
- 推荐组件：
  - `kicker`
  - `h1`
  - `p.big.muted`
  - `blockquote`

封面页要解决“这份演示在讲什么、讲给谁、用什么语气讲”。

### 2. 开场钩子页

- 推荐结构：`section.slide`
- 适用内容：
  - 常见误解
  - 开场问题
  - 路线预告或目录
- 推荐组件：
  - `h2`
  - `ul.emoji`
  - `grid c2`
  - `blockquote`

这类页面负责拉近距离，先让听众点头。

### 3. 概念解释页

- 推荐结构：`section.slide + flow` 或 `section.slide + grid c2`
- 适用内容：
  - 某个核心概念的白话定义
  - 两种方式的对比
  - 一个概念加一个类比
- 推荐组件：
  - `p.big`
  - `flow`
  - `card`
  - `blockquote`

抽象概念优先放进流程节点、卡片或大白话导语，不要直接扔一整段教科书定义。

### 4. 历史或时间线页

- 推荐结构：`table`
- 适用内容：
  - 发展阶段
  - 里程碑
  - 演变过程
- 推荐组件：
  - `table`
  - `blockquote`

如果信息天然带时间顺序，优先使用表格或步骤型结构。

### 5. 多项并列页

- 推荐结构：`grid c3` 或 `grid c2`
- 适用内容：
  - 三个要点
  - 三个工具
  - 三种做法
  - 三个常见错误
- 推荐组件：
  - `card`
  - `card`

并列信息优先拆开，而不是写成一段长列表。

### 6. 术语澄清页

- 推荐结构：表格或先引语后表格
- 适用内容：
  - 三个名词区别
  - 缩写解释
  - 容易混淆的概念
- 推荐组件：
  - `table`
  - `blockquote`

术语类内容以“辨析”为目标，不以“定义齐全”为目标。

### 7. 数据或事实页

- 推荐结构：卡片、表格、列表的组合
- 适用内容：
  - 关键数字
  - 对比指标
  - 结论性事实
- 推荐组件：
  - `card`
  - `table`
  - `blockquote`

不要把大量数字埋进段落。关键数字优先独立成可扫读的模块。

### 8. 结论或金句页

- 推荐结构：`blockquote`
- 适用内容：
  - 一句话结论
  - 价值判断
  - 关键洞察
- 推荐组件：
  - `blockquote`
  - `p.big`

关键结论要让人记住，而不是只让人看完。

### 9. 收尾页

- 推荐结构：`section.slide`
- 适用内容：
  - 总结
  - 下一步建议
  - 行动入口
- 推荐组件：
  - `blockquote`
  - `ul.emoji`
  - `p.muted`

收尾页要给听众一个“我现在可以做什么”的出口。

## 页面节奏规则

- 不要连续多页都使用同一种结构
- 解释页、表格页、卡片页、quote 页要交替出现
- 一份 10-20 页的 deck，至少包含：
  - 1 页封面
  - 2-4 页概念解释
  - 1-2 页表格或时间线
  - 1-3 页并列卡片页
  - 1 页高记忆点结论页
  - 1 页结尾页

## 选型优先级

- 抽象概念优先：`flow` 或 `p.big + blockquote`
- 多项并列优先：`grid c2` / `grid c3`
- 术语澄清优先：表格
- 历史演变优先：表格或流程
- 关键结论优先：`blockquote`
- 操作示例或命令示例优先：`pre > code`

## 不推荐做法

- 不把所有页面都做成“标题 + 一大段正文”
- 不为了花样而切换陌生布局
- 不在一页里同时塞入三个以上层级的主题
- 不把 notes 里讲的内容原样堆回页面正文
